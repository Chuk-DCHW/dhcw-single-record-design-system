Spacing Tokens

A 4px base-unit scale gives every product a consistent spatial rhythm. Primitive tokens define the raw steps; semantic tokens map those steps to purpose. Components reference semantic tokens only.

Accessibility

Touch Target Sizing

Minimum target sizes required by WCAG 2.2 SC 2.5.8.

44 × 44px minimum

WCAG 2.2 SC 2.5.8 (Target Size, AA) — Every interactive element must meet a minimum touch target of 44 × 44px. This is enforced at the component level. Where a visual element is smaller (e.g. a 24px icon button), the surrounding tap/click area must be expanded to meet the minimum via padding or an invisible hit area.

.NET MAUI — Use MinimumWidthRequest / MinimumHeightRequest to enforce the minimum on touch controls.

Blazor — Ensure all interactive elements have a computed height and width of at least 44px via padding or explicit sizing. Do not rely solely on content size.

Do not create custom grid configurations per product. Extend via /products/{name}/ overrides only when genuinely required.
